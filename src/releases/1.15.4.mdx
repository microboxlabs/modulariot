# 🧾 Release Notes – v1.15.4 – Plataforma MBL

**Fecha de entrega**: Por definir\

**Objetivo general**: Esta versión se enfocó en reforzar la experiencia del usuario dentro del expediente tipo Bentobox mediante la implementación del componente de carga y visualización de archivos (drag and drop), así como una vista rápida del estado de validaciones críticas.

Estas mejoras permiten una gestión más accesible, visual e informativa del estado del viaje y sus documentos asociados.

## 🧬 Evolutivos – Nuevas funcionalidades implementadas

### 📂 Implementación del componente Drag and Drop para imágenes y documentos

Se desarrolló un componente visual multifuncional que permite cargar, visualizar y gestionar archivos asociados a un viaje directamente desde el expediente.
Este desarrollo considera los siguientes estados.

**Operativos definidos en Figma**:
- Empty State (sin archivos cargados)
- Default State (vista inicial con archivos visibles)
- Drag and Drop State (al arrastrar archivos)
- Loading Element State (mientras se suben elementos)
- Multiple Elements Upload State (carga simultánea de múltiples archivos)
- Definition of the elements (categorización de los elementos)
- Upload Completion State (transición breve tras la carga exitosa)

**Funcionalidades**:
- Visualización de imágenes y documentos asociados a cada viaje (ej. permisos, documentos firmados).
- Al hacer clic en **“Ver más”** en la sección de imágenes, se abre un **modal** con la galería ampliada.
- Al hacer clic en **“Ver más”** en la sección de documentos, se abre un **modal** con todos los archivos disponibles.
- Al seleccionar un documento PDF específico, este se abre en una nueva pestaña del navegador utilizando el visor estándar de PDF.

**Entregables**:
- Componente funcional y visual completo.
- Integración de modales para imágenes y documentos.
- Visualización ampliada y apertura externa de documentos PDF.

Este componente permite centralizar la evidencia documental del viaje, como permisos,  registros y documentos firmados.

### ✅ Vista rápida de validaciones dentro del Bentobox

Se incorporó una sección de visualización de validaciones clave con un sistema de iconografía intuitivo que permite conocer rápidamente si un punto de control ha sido:

- ✅ Aprobado (ícono check)

- ⚠️ En alerta (ícono exclamación)

**Validaciones incluidas**:
  - GPS
  - Consolidación
  - Separación Documental
  - Validación sistema cliente
  - Fechas asignadas (salida, llegada, entrega)
  - Test de alcoholemia, drogas y somnolencia
  - Validación por App del conductor
  - Validación biométrica

Esto permite al usuario identificar de inmediato si existen condiciones pendientes o incumplimientos relevantes antes de continuar con el flujo.

### 📌 Conclusión

La versión 1.15.4 mejora de forma significativa la interacción con el expediente de viaje, dotándolo de
nuevas   herramientas   para   la   carga   y   visualización   de   archivos,   así   como   una   lectura   inmediata   del estado de validaciones clave.

Estas mejoras fortalecen el control operativo, la revisión documental y la claridad del proceso en curso.
