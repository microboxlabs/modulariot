# Claves Canónicas

Referencia completa de mapeo de claves de métricas desde fuentes OBD-II y J1939.

## ¿Por qué Claves Canónicas?

Los dispositivos reportan datos usando identificadores propietarios:
- OBD-II usa PIDs (Identificadores de Parámetros)
- J1939 usa SPNs (Números de Parámetros Sospechosos)
- Los rastreadores GPS usan nombres de campo propietarios

Las claves canónicas proporcionan una capa de abstracción única que normaliza todos los formatos en nombres consistentes y legibles.

## Mapeo de OBD-II a Canónico

### Métricas de Tren Motriz

| PID | Fórmula en Bruto | Clave Canónica | Unidad | Notas |
|-----|------------------|----------------|--------|-------|
| 0x04 | A × 100 / 255 | `engine.load` | % | Carga calculada del motor |
| 0x05 | A - 40 | `engine.coolant_temp` | °C | Temperatura del refrigerante |
| 0x06 | (A - 128) × 100 / 128 | `fuel.trim.short` | % | Ajuste de combustible a corto plazo banco 1 |
| 0x07 | (A - 128) × 100 / 128 | `fuel.trim.long` | % | Ajuste de combustible a largo plazo banco 1 |
| 0x0C | (A × 256 + B) / 4 | `engine.rpm` | rpm | Valor de dos bytes |
| 0x0D | A | `vehicle.speed` | km/h | Valor directo de byte |
| 0x0F | A - 40 | `engine.intake_air_temp` | °C | Temperatura del aire de admisión |
| 0x11 | A × 100 / 255 | `throttle.position` | % | Posición del acelerador |
| 0x1F | A × 256 + B | `engine.runtime` | s | Tiempo de funcionamiento desde arranque |
| 0x2F | A × 100 / 255 | `fuel.level` | % | Nivel de combustible |
| 0x42 | (A × 256 + B) / 1000 | `battery.voltage` | V | Voltaje del módulo de control |
| 0x5E | (A × 256 + B) / 20 | `fuel.rate` | L/h | Tasa de flujo de combustible del motor |

## Convenciones de Unidades

ModularIoT usa unidades métricas SI consistentes:

| Medida | Unidad Canónica | Alternativas a Convertir |
|--------|-----------------|-------------------------|
| Velocidad | km/h | mph → × 1.60934 |
| Temperatura | °C | °F → (F - 32) × 5/9 |
| Distancia | km | millas → × 1.60934 |
| Volumen | L | galones → × 3.78541 |
| Presión | kPa | psi → × 6.89476 |
| Voltaje | V | mV → ÷ 1000 |

## Guía de Conversión

### Ejemplo: Conversión de RPM de OBD-II

```java
// Datos en bruto: bytes A=0x1A, B=0x0C (PID 0x0C)
int a = 0x1A;  // 26
int b = 0x0C;  // 12

// Fórmula OBD-II: (A × 256 + B) / 4
double rpm = ((a * 256) + b) / 4.0;  // = 1667 rpm

// Crear métrica canónica
MetricItem metric = MetricItem.of("engine.rpm", rpm);
```

### Ejemplo: Conversión de Velocidad J1939

```java
// Datos en bruto del SPN 84: valor de dos bytes en 1/256 km/h
int rawSpeed = 18432;  // bytes de ejemplo

// Fórmula J1939: valor × resolución
double speed = rawSpeed * (1.0 / 256.0);  // = 72.0 km/h

// Crear métrica canónica
MetricItem metric = MetricItem.of("vehicle.speed", speed);
```

## Claves de Extensión

Para métricas propietarias no en el registro canónico, usa el prefijo `x.`:

```
x.teltonika.din1         ← Estado de entrada digital 1
x.queclink.report_type   ← ID de reporte específico de Queclink
x.obd.mode6.test123      ← Prueba de monitoreo abordo personalizada
```

Las claves de extensión:
- Omiten la validación de tipo y límites
- Se registran para análisis futuro
- Permiten datos específicos del proveedor junto con métricas estándar
