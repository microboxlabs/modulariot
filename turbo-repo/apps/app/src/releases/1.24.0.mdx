# 🔔 Release Notes v1.24.0 — ModularIoT

### Fecha: 01/12/2025

**Objetivo**: Fortalecer las capacidades de cálculo predictivo de tiempos de viaje, mejorar la experiencia de usuario en interfaces de monitoreo, ampliar las validaciones de seguridad en la App Conductor, y consolidar la infraestructura de alta disponibilidad con correcciones críticas de estabilidad en servicios backend.

## 🚀 Evolutivos

- **📍 Validación de Instalación de App Conductor**
  - **Verificación automática** durante la preparación del servicio para confirmar que el conductor tiene la app instalada y funcionando.
  - **Integración con el expediente de servicio**: La validación ocurre en el flujo "Presentar Conductor → Controlar Servicio".
  - **Estado de conectividad**: Monitoreo en tiempo real del estado de conexión del conductor y versión de la app.

  - **Sistema de validación en tiempo real** para asegurar que los conductores tienen el tracking de ubicación activado en segundo plano.
  - **Procesamiento de eventos MIOT**: Detección automática del estado `is_location_background` desde eventos de telemetría.
  - **Código de validación `APP_DRIVER`**: Nuevo reason code integrado en el framework de validaciones.
  - **Mapeo inteligente de estados**:
    - `true` → 0 (válido)
    - `false` → 1 (inválido)
    - `missing` → -1 (no encontrado)
  - **Almacenamiento en workflow**: Variable `mintral_appDriverValidation` en tareas de Activiti.
  - **Cumplimiento normativo**: Satisface requerimientos de monitoreo de seguridad y compliance.

- **🧮 Sistema de Cálculo de ETA en Webscript ECM**
  - **Endpoint dedicado** `/alfresco/s/mintral/tasks/calculate-eta` para cálculo de tiempo estimado de arribo.
  - **Cálculo basado en datos históricos**: Utiliza trips previos entre geofences de origen y destino.
  - **Percentiles configurables**: Opciones p50 (optimista), p75 (recomendado), p90 (conservador) para ajustar estimaciones según riesgo.
  - **Ajuste por patrones temporales**: Considera hora del día y patrones de tráfico históricos.
  - **Soporte para conductores dobles**: Reducción de tiempo estimado del 15-20% para viajes con doble tripulación.
  - **Estrategia de fallback**: Cálculo basado en distancia con velocidades promedio cuando no hay datos históricos suficientes.
  - **Caché inteligente**: TTL de 5-10 minutos para optimizar performance en consultas repetidas.
  - **Integración frontend**: API route `/app/api/task/calculate-eta` con hooks React (`useLiveETA()`) para actualización en tiempo real.

- **✨ Mejora UX: Feedback Visual en Botón Refresh de Tabla de Síntomas**
  - **Animación de carga**: El botón refresh ahora muestra una animación de rotación (`animate-spin`) durante la recarga de datos.
  - **Utilización de estado `isValidating`**: Integración con hook SWR para reflejar el estado real de fetching de datos.
  - **Feedback inmediato**: Los usuarios ahora saben instantáneamente cuando sus clicks han sido registrados.
  - **Experiencia mejorada**: Eliminación de clics repetidos por falta de respuesta visual.

## 🔧 Integraciones

- **☸️ Corrección de Alta Disponibilidad en Service Enrichment**
  - **Health checks implementados**: Endpoints `/q/health/live` y `/q/health/ready` en Quarkus.
  - **Configuración de probes en Kubernetes**:
    - Liveness probe para detectar crashes y reiniciar automáticamente pods fallidos.
    - Readiness probe para remover pods no listos del load balancing.
  - **Recuperación automática**: Kubernetes reinicia el servicio sin intervención manual cuando detecta fallos.
  - **Eliminación de downtime**: Los pods se reemplazan automáticamente manteniendo disponibilidad del servicio.
  - **Monitoreo mejorado**: Integración con Alfred Telemetry Platform para visibilidad de salud del servicio.

## 🐛 Correcciones

- **🔧 Reinicio Automático de Service Enrichment en Kubernetes**
  - Corrección crítica de health probes apuntando a endpoint incorrecto (`/` en lugar de `/q/health`).
  - Implementación de liveness y readiness probes funcionales.
  - Eliminación de necesidad de intervención manual para reiniciar pods crasheados.
  - Mejora significativa en disponibilidad del servicio de enriquecimiento.

- **✅ Validaciones de Seguridad en App Conductor**
  - Corrección del sistema de validación de ubicación en background.
  - Implementación de checks automáticos para instalación de app.
  - Prevención de asignación de servicios a conductores sin herramientas adecuadas.

- **🔄 Migración Next.js 14.x → 16.x con React 19**
  - Actualización completa del framework frontend para aprovechar mejoras de performance y React 19.
  - Modernización de dependencias (Flowbite React 0.12.10, Next Auth 5.0.0-beta.30).
  - Compatibilidad con características concurrentes de React 19.
- **🎨 Badges de Origen/Destino en Kanban Cards**
  - Sistema de iconografía visual para identificar tipo de servicio (Cliente, Proveedor, Mintral).
  - 7 combinaciones de badges con código de colores para scanning rápido.
- **📊 Dashboard SPC - Componentes Expandidos y Tracking de Jaulas**
  - Nuevo sistema de tracking para dispositivos tipo "jaula".
  - Tablas expandidas con información completa de servicios.
  - Componente de mapas con tracking en tiempo real.
- **📱 Formulario Dinámico para Cálculo de ETA en MonitorTrip**
  - Dos modos de operación: calculado automático vs manual.
  - Integración con webscript de cálculo de ETA.
- **⚙️ Optimizaciones de Performance Backend**
  - Refactorización de PgRestService con abstracción HttpService.
  - Query SQL optimizado para estadísticas de tareas.
- **🔐 Firma Digital POD con Validación NFC y Biométrica**
  - Implementación de prueba de entrega digital con validación avanzada.
- **🏢 Mejoras App Tótem de Recepción**
  - UX mejorada y calidad de señal GPS optimizada.
- **📋 Flujo de Validación POD con Integración Diana y Alerce**
  - Cierre de viajes mejorado con integración de sistemas externos.
