# Deployment

Infrastructure requirements and deployment options for ModularIoT.

## Deployment Models

### Cloud (SaaS)
Fully managed ModularIoT service:
- No infrastructure to manage
- Automatic updates
- Built-in redundancy
- Consumption-based pricing

### Dedicated Cloud
Isolated cloud deployment:
- Tenant-specific infrastructure
- Enhanced security
- Custom configurations
- Managed by ModularIoT

### On-Premises
Self-hosted deployment:
- Full infrastructure control
- Data locality requirements
- Air-gapped options
- Customer managed

## Infrastructure Requirements

### Compute
- Container orchestration (Kubernetes)
- Autoscaling capability
- Multi-zone distribution

### Storage
- Time-series database
- Object storage
- Distributed caching

### Networking
- Load balancing
- TLS termination
- Network segmentation

## High Availability

- Multi-zone deployment
- Automatic failover
- No single points of failure
- Disaster recovery procedures

For self-hosted deployments, see the installation guide (available to licensed customers).
