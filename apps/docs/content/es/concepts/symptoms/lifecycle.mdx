# Lifecycle

Symptoms follow a defined lifecycle from detection through resolution.

## Lifecycle Stages

```
Detection → Notification → Acknowledgment → Resolution → Closure
```

## Detection

The platform identifies a condition:
- Rule conditions are met
- Symptom instance created
- Initial severity assigned
- Subject (asset) linked

## Notification

Stakeholders are informed:
- Push notifications sent
- Dashboard updated
- Worklist populated
- Escalation timers started

## Acknowledgment

Operator indicates awareness:
- State changes to acknowledged
- Accountability assigned
- Notification escalation paused

## Investigation

Operator investigates:
- Reviews symptom details
- Examines related data
- Determines root cause
- Plans response

## Resolution

Condition is addressed:
- Problem fixed
- Or condition clears naturally
- Resolution action recorded
- State changes to resolved

## Closure

Record finalized:
- Full audit trail preserved
- Metrics updated
- Learning captured
- Historical record complete
