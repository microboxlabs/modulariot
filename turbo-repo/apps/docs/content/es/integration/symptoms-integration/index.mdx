# Symptoms Integration

Learn how to define custom symptoms and integrate with the Symptoms Engine outputs. This section bridges the gap between platform capabilities and your operational workflows.

## Overview

The Symptoms Engine provides powerful detection capabilities out of the box. This section covers how to extend and customize symptom behavior for your specific use cases.

## Topics

### Defining Symptoms
Create custom symptom definitions that match your operational requirements and domain knowledge.

### Custom Metrics
Define custom metrics that feed into symptom detection rules.

### Consuming Outputs
Integrate symptom notifications into your existing systems and workflows.

## Integration Points

- **Webhook notifications** - Real-time symptom events to your systems
- **API polling** - Query symptom state on demand
- **Event streaming** - Subscribe to symptom event streams

## Use Cases

- Custom fleet health monitoring
- Predictive maintenance triggers
- Compliance and safety alerting
- Operational SLA tracking
