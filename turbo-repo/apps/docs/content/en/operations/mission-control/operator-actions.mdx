# Operator Actions

Standard workflows for responding to symptoms.

## Action Types

### Acknowledge
Indicate awareness of symptom:
- Stops escalation timer
- Records who acknowledged
- Updates symptom state

### Assign
Designate responsible operator:
- Clear accountability
- Appears in assignee's worklist
- Can reassign if needed

### Investigate
Research the symptom:
- Review telemetry data
- Check asset history
- Identify root cause

### Respond
Take corrective action:
- Contact driver/operator
- Dispatch service
- Remote command
- Document action

### Resolve
Close the symptom:
- Mark as resolved
- Record resolution type
- Add resolution notes

## Action Logging

All actions are logged:
- Who performed action
- When action taken
- What was done
- Related notes

## Standard Operating Procedures

Link symptoms to SOPs:
- Symptom type → SOP mapping
- Guided response steps
- Required documentation
- Escalation triggers

## Bulk Actions

Handle multiple symptoms:
- Bulk acknowledge
- Bulk assign
- Bulk resolve (with care)
