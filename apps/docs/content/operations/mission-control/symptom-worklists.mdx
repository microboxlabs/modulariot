# Symptom Worklists

Organized queues of symptoms requiring operator attention.

## Purpose

Worklists organize symptoms into actionable queues:
- Prioritized for efficient handling
- Assigned for accountability
- Tracked for completeness

## Default Worklists

### All Active
Every unresolved symptom across the fleet.

### My Assigned
Symptoms assigned to current operator.

### Critical
All critical and emergency severity symptoms.

### Unacknowledged
Symptoms awaiting initial response.

## Worklist Columns

Standard worklist displays:
- Severity indicator
- Symptom type
- Asset identifier
- Detection time
- Current state
- Assigned operator

## Filtering

Narrow worklist by:
- Severity level
- Symptom type
- Asset group
- Time range
- Assignment status

## Actions

From worklist, operators can:
- View symptom detail
- Acknowledge symptom
- Assign to operator
- Add notes
- Mark resolved

## Custom Worklists

Create custom worklists for:
- Specific symptom types
- Asset groups
- Shift handoffs
- Escalation queues
