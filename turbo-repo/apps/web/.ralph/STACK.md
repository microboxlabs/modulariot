## 2. Tooling configs to copy verbatim into `apps/web/`

### `tsconfig.json`
- Extends `@repo/typescript-config/nextjs.json`
- Paths: `"@/*": ["./src/*"]`, `"@assets/*": ["./public/*"]`
- `compilerOptions.types`: include `["vitest/globals"]` if vitest is used
- Compiler deltas to keep: `target: ES2017`, `jsx: react-jsx`,
  `lib: ["dom","dom.iterable","esnext"]`, `noUncheckedIndexedAccess: false`

### `eslint.config.mjs`
- FlatConfig array; ignores first, then spread `nextJsConfig` from `@repo/eslint-config/next-js`
- Ignore set: `.next/**`, `node_modules/**`, `coverage/**`, `*.js`, `public/**`

### `next.config.mjs`
- Plugins, in order: `withFlowbiteReact(withMDX(nextConfig))` â€” flowbite-react ships its
  own Next plugin in v0.12+ which must wrap the config
- `pageExtensions: ["ts","tsx","js","jsx","md","mdx"]`
- `outputFileTracingRoot`: monorepo root (npm workspaces tracing)
- `productionBrowserSourceMaps: true`
- **DIFFERENCE for apps/web**: do NOT set `basePath: "/app"`. Marketing site lives at root.
- **DIFFERENCE**: drop the Azure Blob `images.remotePatterns` (apps/app-specific media).
- **DIFFERENCE**: drop `serverExternalPackages: ["pino","pino-pretty"]` and
  `transpilePackages: ["@microboxlabs/miot-calendar-client"]` â€” neither applies to web.

### `postcss.config.mjs`
- Single plugin: `@tailwindcss/postcss`. Copy verbatim.

### Tailwind v4 setup
- `@import "tailwindcss"` in `globals.css` (no `@theme` block in apps/app today)
- `tailwind.config.ts` only extends animations / keyframes (show/hide/fade/orbit/carousel-progress)
- **OPPORTUNITY**: apps/app has NO brand color overrides. P0-04 will introduce them
  via a `@theme` block in `globals.css` (Tailwind v4 native) â€” using CSS custom
  properties so flowbite-react can pick them up. This is greenfield, not copy.

### `.prettierrc`
- `semi: true, trailingComma: "es5", singleQuote: false, printWidth: 80, tabWidth: 2`

### `knip.json`
- Entry: `src/app/**/*.{js,ts,jsx,tsx}`, `src/middleware.{js,ts}`, config files
- Project: `src/**/*.{js,ts,jsx,tsx}`
- Ignore: tests, `.d.ts`, `src/releases/**`
- ignoreDependencies: `eslint`, `@types/*`, `prettier`, `tailwindcss`, `autoprefixer`, `postcss`

## 4. Theme + dark mode pattern (copy verbatim)

- `apps/app/src/features/theme/components/ThemeDetector.tsx` â€” wraps children in
  `<ThemeProvider theme={flowbiteTheme}>`. Detects via `matchMedia` + cookie sync.
- `apps/app/src/features/theme/flowbite-theme.ts` â€” minimal `createTheme()` overriding
  `button.base` and `sidebar.root.inner`. apps/web should DELETE the sidebar override
  (no sidebar) and ADD bento-card / navbar / hero-button overrides as Phase 0â€“2 produce them.
- `apps/app/.flowbite-react/init.tsx` (auto-generated) â€” `<ThemeInit />` mounted in root layout.
- `apps/app/.flowbite-react/config.json` â€” `{ dark: true, rsc: true, tsx: true, version: 4 }`.
  Copy this file as-is when bootstrapping apps/web.
- Root layout: `<ThemeModeScript />` in `<head>`, `<ThemeInit />` + `<ThemeDetector>{children}</ThemeDetector>` in `<body>`.

## 5. Layout primitives to mirror

- Font: `Inter` from `next/font/google`, subset `["latin"]`. apps/app applies it via
  `twMerge(inter.className, ...)` on the secured layout's wrapper. apps/web should
  apply it on the `<html>` or root `<body>` directly (no nested layouts at marketing site).
- Metadata: `export const metadata` in `app/layout.tsx`, env-driven title/description.
  apps/web: hard-code defaults, env override optional.
- Public assets: `apps/web/public/brand/` (rescued from apps/web-site in P0-02).
- Toaster: Sonner is wired in apps/app; marketing site likely doesn't need it for v1.
  Skip until a form lands (e.g., demo-request).

## 7. `package.json` scripts to mirror

```json
{
  "dev": "next dev --turbopack --port 3041",
  "build": "NODE_OPTIONS=--enable-source-maps next build",
  "start": "next start",
  "lint": "eslint .",
  "lint:fix": "eslint . --fix",
  "check-types": "tsc --noEmit",
  "format": "prettier --write .",
  "format:check": "prettier --check .",
  "test": "vitest",
  "test:run": "vitest run",
  "knip": "knip",
  "knip:check": "knip --no-exit-code",
  "postinstall": "flowbite-react patch"
}
```

Notes:
- Port `3041` chosen to avoid clashing with apps/app (`3050`) and apps/web-site (`3040`).
- `--turbopack` flag matches apps/web-site's dev script (faster HMR for marketing iteration).
- `postinstall: flowbite-react patch` â€” present in apps/web-site, NOT in apps/app.
  v0.12+ docs recommend it; include it.
- `NODE_OPTIONS=--enable-source-maps` on build â€” matches apps/app, useful for prod debugging.

## 8. What to explicitly NOT copy from apps/app

- Auth context (`features/auth/context/auth-context.tsx`) + next-auth + SessionProvider
- API route handlers under `app/api/*` (BFF / tenant-scoped server actions)
- i18n routes (`[lang]` dynamic segment, `ParamsWithLang`)
- Domain features: `dashboard`, `signal-history`, `symptoms`, `map-visualization`,
  `fleet-management`, `shipping`, `calendar`, `where-is-my-load`

  `html2canvas-pro`, `lottie-react`, `@alfresco/js-api`, SSE infrastructure
- Vitest setup file references (write a minimal one for apps/web instead)
